🎯 基于位次的录取概率算法说明

核心理念：位次比分数更准确！即使你考了700分，但如果有10000个人考了701分，你也可能被挤出录取线。

📊 算法核心逻辑

位次差距计算公式

位次差距 = 用户位次 - 学校历年最低录取位次

注意：位次越小越好，负值表示用户位次更优秀

🔄 算法优化对比

❌ 旧方法：基于分数

  • 只看分数差距
  • 忽略考生竞争激烈程度
  • 不考虑分数分布
  • 容易误判录取概率

问题：700分vs701分看起来差1分，但可能位次差几千名

✅ 新方法：基于位次

  • 直接反映竞争激烈程度
  • 考虑考生分布密度
  • 更准确的录取预测
  • 符合高校录取实际

优势：位次直接对应录取名额，更加精准

🎯 为什么位次更准确？

高考录取的本质

高校录取本质上是按位次录取，而不是按分数录取：

  • 招生计划固定：每个学校每年招生人数基本固定
  • 位次对应名额：位次直接对应录取名额
  • 分数会波动：每年试题难度不同，分数会波动
  • 位次相对稳定：位次反映的是相对排名，更稳定

举例说明

某大学在河南招100人：

  • 2023年：最低分650分，对应位次25000名
  • 2024年：最低分630分，对应位次24800名

分析：虽然分数降了20分，但位次基本稳定，说明录取难度没变

⚙️ 算法实现细节

双重保障机制

  1. 优先使用位次：如果有位次数据，优先基于位次计算
  2. 分数作为备选：如果没有位次数据，使用分数计算
  3. 智能降级：确保在任何情况下都能给出合理建议
💡 使用建议：
  • 重点关注位次差距，而不是分数差距
  • 保底学校选择位次优势明显的
  • 冲刺学校可以选择位次略有不足的
  • 稳妥学校选择位次接近的
